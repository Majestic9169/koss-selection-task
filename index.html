ABOUT BOOTING


						1. What happens when a computer is turned on
							a. What is a bootloader (file that loads kernel)
							b. What is a kernel (makes hardware usable)
						2. What are the differences in Legacy BIOS and uefi
						3. Why did I change from UEFI to BIOS
					
this is your computer when turned off

						Turning on your computer gives power to your 
						Motherboard and other hardware and triggers the 
						BIOS chip on your motherboard
					

						The BIOS chip contains your firmware
						First it triggers the Power-On Self-Test
					

						Here the motherboard verifies that all required hardware
						is present and operational
					
						
							After POST the BIOS chip looks for a boot device
							this could be your SSD, HDD, external USB, and even
							ethernet
						
					

						Then the BIOS chip runs the bootable file on the device
						This is either the MBR or an efi file
						The first thing on the bootfile is usually the bootloader
						in most Linux distros the bootloader is GRUB2
						What the bootloader does is load the kernel into memory
					
				

						What is the Kernel?
						The kernel is a layer of software that sits between the 
						hardware and the operating system
						It is what loads your device drivers and makes your 
						hardware usable
					

						Imagine you are the software
						and TSC is the hardware
						that makes your football equipment the kernel, it is what 
						enables you make use of the football field properly

BIOS: Basic Input-Output System
[] Just a specification about how to boot
[] Older, not for drive space >2TB
[] Uses Master Boot Record to load bootloader
UEFI: Unified Extensible Firmware Interface
[] Successor of BIOS
[] Uses Guid Partition Table to find and load bootloader
[] A little faster 

So why did I switch to Legacy BIOS?
						
							Even though I was using UEFI and everything was correct, 
							my laptop couldn't detect the correct efi file
							Because of this I had to manually select my efi
							file during boot
							I tried fixing this issue by changing the boot order using
							efibootmgr, but that didn't help
							Finally I switched to BIOS and ended up on my current setup
							An easy fix would be to create a custom efi boot path in 
							the UEFI settings, however I discovered the wiki article 
							much later when I was looking up how to setup tlp

						Graphical User Interface

						  for X11, however it does not have as much support yet
